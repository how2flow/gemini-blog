## 블로그 일상 작성

- 문서 생성은 다음의 단계를 거칩니다.
프로젝트의 컨벤션 기반으로 작성되어야 합니다.
한글로 작성되어야 합니다.

./ 트리 구조(예시)는 다음과 같습니다.

```
.
├── 2025
│   └── book
│       ├── 2025-01-31-trendko.md
│       ├── 2025-02-28-rich.md
│       ├── 2025-03-31-here.md
│       └── 2025-04-30-dual-brain.md
├── 2025-book.md
...
```

jekyll collection 이기 때문에 jekyll의 포스팅 규칙에서 자유롭습니다.
규칙에서 자유롭습니다만, 날짜 단위로 관리를 하고 싶기 때문에, 기존 컨벤션 기반으로 작성합니다.
연도에 맞는 디렉토리를 만들고 그 아래에 md 파일로 작성합니다.
같은 연도에 주제별로 또 묶입니다.
2025년도의 경우에는 독후감만 업로드 했지만, 2026년도나 추후 다른 주제도 추가될 수 있기 때문에 해당 컨벤션 기준으로
추가 작성해야 할 수 있습니다. (25년도 게시도 추가할 수 있습니다.)


../assets/images/diary/ 아래 트리 구조(예시)는 다음과 같습니다.

```
.
└── book
    ├── 01-ko.jpg
    ├── 02-rich.jpg
    ├── 03-hero.jpg
    └── 04-brain.jpg
...
```

1. 문서는 실시간성을 띄지 않습니다. 사용자가 언급이 없으면 실시간 기준으로 작성하되, 사용자가 기간을 명시하면 해당 기간을 기준으로
문서를 작성합니다.

2. diary는 ```../_pages/diary.md``` 에서 먼저 diary 연도와 주제 별로 이동해서 볼 수 있게 설계되어 있습니다.
따라서, 주제가 추가 될 때는 _pages의 내용도 함께 업데이트가 되어야 합니다.

3. ```diary``` 안에서도 연도-주제.md가 함께 업데이트 되어야 합니다.

4. 종류별 게시 방법:
4-2. 일상: 일상 일기 쓰듯 작성해 주세요. 2030대의 평범한 일상에 대한 내용입니다. 사용자가 요청한 것을
자연스럽게 문장을 만들어 주고 문단과 문단 사이에도 자연스럽게 흐름이 이어지도록 작성해주세요.
4-1. 독후감: 기존 독후감 게시글들을 참조하여 비슷한 스타일로 작성해주면 됩니다. 정석적인 독후감 글의
흐름대로, 간단하게 어떤 내용이고, 어떤 느낌을 받았고 등등..
4-3 기타: 또 다른 특별한 주제가 있다면 아마 룰에 새로 추가를 하겠지만, 없다면 GoogleSearch를 통해 다른
게시글들을 확인하면서 비슷한 흐름으로 글을 작성해주세요.

5. 사진
사진 생성을 하지 않습니다. 일상의 사진들은 제가 직접 찍은 사진들을 assets에 추가할 예정입니다.
사용자가 언급을 따로 하거나, git status 를 통해 새로 추가된 사진이 있으면, 그 사진을 사용하여 글을 작성합니다.

6. 필요한 정보가 있다면 GoogleSearch 도구를 사용하여 추가 설명을 해주어야 합니다.
search.md 파일에 검색한 내용을 작성합니다.
GoogleSearch 도구를 사용해야 하며, url도 함께 작성되어야 합니다.

7. 초안을 draft.md 파일에 작성합니다. 그리고, draft.md 파일을 기반으로 게시글을 작성합니다.

```
---
permalink: /diary/2025/...
title: ""
excerpt: ""
header:
  teaser: /assets/images/diary/...
date: yyyy-mm-dd
layout: single
---
```
으로 문서는 시작합니다.

8. 중복에 대한 문제..
작업 중에, path와 이름이 중복이 된다면 images의 경우 overwriting이 될 수 있습니다. 이럴 경우, 반드시 사용자에게 경고를 전달하여야 하며
overwriting은 허용될 수 없습니다.

8. 이미지 생성에 대한 문제..
이미지 생성을 하게 될 경우, gemini-image-gen 도구를 반드시 사용해야 하며,
이미지 생성이 실패가 되었을 경우, (0kb) 사용자에게 즉시 보고해야 하며, 모든 동작을 중단해야 합니다.
error 보고를 반드시 해야 합니다.

9. 문서 형식에 대한 규칙
문장 최대 길이는 120자를 넘지 않게, 요청 드립니다. jekyll 구조 상 toc가 있기 때문에 너무 길 필요가 없습니다.
줄바꿈은 ```<br>``` html 커멘드로 문서 편집을 하는 사용자도 구조가 잘 보이게끔 편집해주세요.

10. 용어에 대한 강조
이 프로젝트는 span 을 사용한 용어 강조를 지원하고 있습니다.

사용 예시:
```
Server <span style="{{ site.code }}">Host</span> : ...
```

위와 같이 span을 쓰고, 종료할 때에는 반드시 한 칸을 띄워야 합니다.
